# AGENTS.md

## Docs rules
- Keep design docs, implementation docs, and operational docs distinct.
- Do not describe aspirational behavior as if it is already implemented.
- When contracts, storage behavior, or local-first assumptions change, update the relevant docs in the same round.
- Prefer documenting invariants, boundaries, and tradeoffs over marketing language.
- Large implementation changes should link to a concrete plan file under `plans/`.
- When AI provider coverage changes, update the provider matrix, auth assumptions, and compatibility notes in the same round.
- Distinguish clearly between direct model vendors and routed model platforms.
